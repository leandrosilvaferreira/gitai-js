### Skill Content

The skill provides comprehensive guidance covering:

#### Pre-Release Checklist (6 checks)

1. No uncommitted changes: `git status --porcelain`
2. On correct branch (main): `git branch --show-current`
3. Tests pass: `npm run test`
4. Build clean: `npm run build`
5. Version not already published: `npm view @notyped/gitai version`
6. Tag doesn't already exist: `git tag -l v$(node -p "require('./package.json').version")`

#### Release Execution

- Single command: `npm run release`
- Explains what the script does internally:
  - Runs tests, builds, executes release flow
  - Prompts for release type (patch/minor/major/custom)
  - Generates release notes via AI
  - Updates package.json, src/version.ts, CHANGELOG.md
  - Creates annotated git tag with release notes
  - Prompts for push

#### Post-Release Verification

- GitHub Release created automatically by CI (reads from annotated tag)
- npm publish automated by CI
- Verification command: `npm info @notyped/gitai`

## Key Design Decisions

1. **Based on actual code:** All checklist items and workflow steps derived directly from `scripts/release-flow.ts` and `package.json`
2. **Annotated tag pattern:** Documented why the release notes are stored in the git tag (so CI can publish without API keys)
3. **Pre-validated flow:** Checklist ensures all preconditions pass before `npm run release` runs
4. **Clear ownership:** Release notes generated by AI, but pushed by CI — no API keys needed in CI pipeline
5. **User-invocable: false:** Skill is not meant to be directly invoked; it guides other agents
